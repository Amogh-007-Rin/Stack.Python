# Exercises: Choosing the Right Data Structure

## Exercise 1: Fixed Coordinates
You need to store a 2D point (x, y) that will never change. Which data structure do you choose? Why?

## Exercise 2: Unique Visitors
You need to track which IP addresses have visited your website today. You only care about uniqueness. Choose a data structure.

## Exercise 3: Student Grades
You need to store student names and their grades. You'll frequently look up a student's grade by name. Choose the best structure.

## Exercise 4: Shopping List
You need a list of items to buy at the store. The list changes frequently (add/remove items). Order matters. Choose the best structure.

## Exercise 5: Finding Duplicates
Given a large list of numbers (100,000 items), find all duplicates. Why is a set better than a list for this?

## Exercise 6: Counting Word Frequency
Write a function `word_freq(text)` that counts how many times each word appears. Which data structure do you use?

## Exercise 7: Queue Simulation
You're simulating a printer queue. Jobs arrive and are processed in order. Choose the right structure.

## Exercise 8: Lookup Speed
Explain why checking membership with `in` is faster in a set than in a list. Include time complexity in your explanation.

## Exercise 9: Configuration Settings
You need to store application settings (key-value pairs) that rarely change. Choose the best structure.

## Exercise 10: Matrix Operations
You need to store a 3x3 matrix of numbers for mathematical operations. Which structure(s) would work? Discuss tradeoffs.
