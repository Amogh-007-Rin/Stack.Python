# Module 056: Error Handling — Quiz

1. **What happens when an exception is not caught?**
   - a) The program continues
   - b) The program terminates with a traceback
   - c) The exception is ignored
   - d) Python logs it silently

2. **What does the `else` clause in `try/except/else` do?**
   - a) Always runs after try
   - b) Runs only if no exception occurred
   - c) Runs only if an exception occurred
   - d) Replaces `finally`

3. **What does the `finally` clause guarantee?**
   - a) It runs only on success
   - b) It runs only on failure
   - c) It always runs, regardless of exceptions
   - d) It runs before the `try` block

4. **Which of these is NOT a built-in exception?**
   - a) `ValueError`
   - b) `TypeError`
   - c) `CustomError`
   - d) `FileNotFoundError`

5. **What does a bare `except:` clause catch?**
   - a) Only `Exception` subclasses
   - b) Only `BaseException` subclasses
   - c) All exceptions including `KeyboardInterrupt` and `SystemExit`
   - d) Only `ValueError`

<details>
<summary>Answers</summary>
1-b, 2-b, 3-c, 4-c, 5-c
</details>
