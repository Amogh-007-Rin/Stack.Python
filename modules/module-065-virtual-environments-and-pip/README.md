# Module 065: Virtual Environments and pip

- **Phase:** 7. Modules, Stdlib & Testing
- **Duration:** 1.5 hours

## Learning Objectives

- Understand why virtual environments are needed
- Create and manage venvs with `python -m venv`
- Install, uninstall, freeze, and list packages with pip
- Work with `requirements.txt` for reproducible environments
- Compare pip vs conda briefly

## Topics Covered

- What venv is and why it matters (dependency isolation)
- `python -m venv <name>` — creating a virtual environment
- Activating/deactivating on Linux, macOS, and Windows
- `pip install`, `pip uninstall`, `pip freeze`, `pip list`
- `requirements.txt` format and generation
- Best practices: pinning versions, using `requirements.txt` in projects
- Brief comparison: pip vs conda
